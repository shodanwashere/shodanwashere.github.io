What is Spring?

Well, it’s a bit confusing. Spring could mean:

  • The Spring Framework;
  • Spring Boot;
  • Spring Data;
  • Spring Cloud;
  • Spring Batch;
  • etc.

Most often, when people say “Spring”, they mean the entire family of projects.

Meet the Spring Family

It all started with the Spring Framework. It aimed at removing complexity and help make things like web development and data access easier to implement. It also strives to avoid creating boilerplate code.

The Spring Framework led to the creation of several other projects built on top of the Spring Framework that give it additional functionality.

Spring Boot makes creating Spring based applications faster and easier, removing the need to configure and choose components by default.

Why Spring?

Spring started as a response to the complexity of Java EE. Today, Spring is so much more than an alternative to Java EE. In fact, it’s complementary to it. Spring makes it easy to create Java enterprise applications.

  • It’s flexible, modular and backwards compatible.
  • Large and active community.
  • Continually innovates and evolves.

Getting to Know Spring with Spring Boot

Getting started with any technology can be confusing and frustrating. Spring Boot makes getting started with Spring quick and easy.

It can be used to build both Web Applications and Non-Web Applications.

It has a number of key features that make it unique and easy to use:

  • Auto-configuration It attempts to automatically configure your Spring application based on the dependencies that you have added.
    • Auto-configurations are contextually aware and smart! For example, if your Spring Application depends on some database-related library, then it will take that context and auto-configure the application for database access.
    • Setting up Auto-configuration is Effortless!
      @EnableAutoConfiguration
public class DemoApplication {
...
}
  • Spring Boot makes it easy to create standalone, production grade, Spring-based applications that you can ‘just run’.
    • Typical Process for Running Java Web Applications:
      • Package the application
      • Choose & Download Webserver
      • Configure Webserver
      • Deploy Application & Start Webserver
    • Process for Running Spring Boot Applications:
      • Package the application
      • Run! Like java -jar my-application.jar
  • Takes an opinionated view of building production-ready Spring applications. Not a nagging type of opinionated, but it favors convention over configuration and is designed to get you up and running as quickly as possible.

Understanding Spring’s Foundations: The Spring Framework

Spring is a Software Framework.

(…) a universal, reusable software environment that provides aparticular functionality as part of a larger software platform to facilitate development of software applications, products and solutions

The Spring Framework was the beginning of Spring. Enterprise Java Development at the time was a complex process. The Spring Framework has grown and evolved over time.

In addition to simplifying Java development, it became the foundation for Spring Projects, which Applications can use to further simplify development.

The Six Key Areas of the Spring Framework

The Spring Framework utilizes a modular architecture. It can be decomposed into 6 key areas:

Core

The most important and foundational module. It provides a number of different features:

  • i18n internationalization support;
  • Validation support;
  • Data binding support;
  • Type conversion support;
  • etc.

At the center of Spring Core is Dependency Injection. During OOP, Developers create objects to represent or model real-life things, but objects don’t exist by themselves (for the most part), they are dependent on other objects.

public class Computer { // a computer doesnt exist by itself
	HardDisk hdd;         // it needs a hard drive
  Memory[] memory;      // and memory to work
}

Dependency Injection is about dealing with the way objects fulfill their dependent objects.

Fulfilling dependencies can be done in one of two ways:

  1. The object fulfills its own dependencies.
    1. Seems easy
    1. Creates tightly coupled objects
  1. The object declares what other objects it depends on and some external entity fulfills those dependencies.
    1. More flexible
    1. Creates loosely coupled objects

Spring Core is a dependency injection container:

  • It creates and maintains objects and their dependencies
    • Less for developer to manage
  • It’s the “glue” of the application.

Web

It’s a framework for handling web requests, with two different Spring submodules

Spring Web MVC depends on a key component called a “Servlet”:

an object that receives a request and generates a response based on the request.

Challenges with the Servlet API:

  • Somewhat a low-level API
    • Not as easy to use
    • Less productive
  • Easy to mix-up concerns

With Spring MVC:

Notice how Spring Web MVC sits behind the Business Logic and ahead of the Servlet API.

  • It’s a higher level API
    • Easier to use
    • More productive
  • There’s a clear separation of concerns:

Spring Webflux

Spring Webflux is based on Reactive Programming:

a declarative programming paradigm concerned with data strams and the propagation of change

In reactive programming, you React rather than wait.

  • Has a different way of handling web requests
    • Asynchronous execution
    • Doesn’t block (wait)
      • Better resource utilization

AOP

(also known as Aspect-Oriented Programming)

A programming paradigm that aims to increase modularity by allowing the separation of cross-cutting concerns.

Without AOP, solving concerns that are cross cutting, results in scattered and duplicated code across many parts of an application.

It’s used to:

  • Implement features in Spring
  • Handle cross-cutting concerns

Example (Application Security):

public void sensitiveOperation(){
	// check if user is authenticated and has the correct role
	if (...) {
		// do sensitive operation
	} else {
		// raise an unauthorized error
		// log failed attempt
    // redirect user to login page
	}
}

Example (Application Security with Spring AOP):

@PreAuthorized("hasRole('admin')")
public void sensitiveOperation(){
	// do sensitive operation
}

Data Access

Almost every application interacts with data. Data is really important.

The Spring Framework’s Data Access module makes it easier to develop applications that interact with data.

Example (code to retrieve data using JDBC):

try{
	Statement stmnt = conn.createStatement();
	try {
		ResultSet rs = stmnt.executeQuery("SELECT COUNT(*) FROM foo");
		try {
			rs.next();
			int cnt = rs.getInt(1);
		} finally {
			rs.close();
		}
	} finally {
		stmnt.close();
	}
} catch (SQLException e) {
	// handle error
} finally {
	try {
		conn.close();
	} catch (SQLException e) {
		// handle error
	}
}
int cnt = new JdbcTemplate(ds).queryForInt("SELECT COUNT(*) FROM foo");

Database transactions with the Spring Framework’s Data Access module are really easy.

try {
	connection.setAutoCommit(false);
	
	// 1 or more queries or updates
	
	connection.commit();
} catch (Exception e) {
	connection.rollback();
} finally {
	connection.close();
}
@Transactional
public void operation(){
	// 1 or more queries or updates
}

The Data Access module also provides a small but useful feature called “Exception Translation”. Different Vendors use different exceptions, but using Data Access’ Exception Translation, vendor-specific exceptions are mapped into a well-known set of exceptions.

For example, for a data integrity violation error:

  • MySQL will return error code 630
  • Oracle DB will return error code 1400
  • Other database engines will return different error codes
  • Spring Data Access will map all of these error codes to the DataIntegrityViolationException.

Testing data is easier with the Spring Framework’s Data Access module.

  • Easy to switch configurations for testing

Integration

It’s all about making different systems and applications work together. How does one application “talk” to another?

This is a multisided problem:

  1. How do you expose operations to other systems?
  1. How do you invoke or run operations on other systems?

There’s many ways to expose operations:

  • RMI (Remote Method Invocation)
  • Messaging Systems
  • Web Services

The Spring Framework makes it easy to both expose and invoke web services.

@RestController // Denotes we're using REST
public class AccountController {
	@GetMapping("/account/{id}")  // denotes operation and path
	public Account find(@PathVariable int id){ // associates path and value
		// look up account by id
	}
}

Invoking a web service is also easy and is made via a Rest Template:

  • It abstracts away tedious details
    • Connecting to the web service
    • Sending the command
    • Handling the response
restTemplate.getForObject("http://path/to/web/service", Object.class);

Testing

Testing is an important part of software development, and there’s different ways to test. Spring Testing focuses on Unit testing and Integration testing.

Unit Testing

a software development process in which the smallest testable parts of an application, called units, are individually and independently scrutinized for proper operation

Explicit support for unit testing is minimal.

  • Benefits come from using dependency injection

When performing unit testing, you want to test the smallest unit of code possible.

  • Dependencies cause challenges.

Dependency injection forces the developer to declare dependencies. During testing, dependent code can be replaced with code that behaves a certain way, i.e., you control how the dependencies are supposed to behave.

This way you:

  • Only test the code, not its dependencies
  • Mock the dependencies

Example (Using Dependency Injection in Testing):

public Permission getPermissions(Security Manager sm) {
	String securityLevel = sm.getSecurityLevel();

	if (securityLevel.equals("CONFIDENTAIL")){
		// give user confidential permissions
	} else if (securityLevel.equals("SECRET")){
		// give user secret permissions
	} else if (securityLevel.equals("TOP_SECRET")){
		// give user top secret permissions
	}
}

Objective: test every condition

Method: (Mock) Control how the SecurityManager (the dependency) behaves to control the test

The Spring Framework comes with several built-in mocks to make testing easier and faster.

Integration Testing

the phase in software testing in which the individual software modules are combined and tested as a group. It occurs after unit testing.

Provides support for piecing together parts of an application for testing

  • Inject dependencies and put everything together

Provides support for common testing scenarios

  • Testing with data
  • Web application testing

Provides support for cleaning up after tests

  • Tests modify things, need to reverse modifications after test so as to not affect the next tests

Is Spring a Good Fit?

The Advantages of Using Spring

  • Rock-solid & well engineered
  • Stood the test of time
  • Has a huge community
  • Well-liked
  • Large talent pool
  • Wealth of existing knowledge
  • Very actively developed (the project is open-source)
  • Built-in IDE support
  • Scalable

The Disadvantages of Using Spring

  • Too much “magic”
  • Steep learning curve
  • Increases the size of your deliverable
  • Hard to debug
  • Adds memory overhead
  • Spring’s complexity has grown over time
  • Spring can be too configurable
  • Spring is “big” (Not good for small problems that need quick results)
  • Spring’s community projects are hit or miss
